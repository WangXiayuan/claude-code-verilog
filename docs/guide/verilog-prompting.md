# Verilog 提示词技巧

掌握针对硬件描述语言的 Prompt 模板，让 Claude Code 输出更高质量的 Verilog 代码。

## 基本原则

### 1. 明确接口约束

含糊的描述 → 随意的接口。清晰的描述 → 规范的接口。

```markdown
# ❌ 不好
帮我写一个 FIFO

# ✅ 好
写一个同步 FIFO，参数化位宽（DATA_WIDTH）和深度（DEPTH，必须是 2 的幂），
包含 full、empty、almost_full、almost_empty 标志，
使用读写指针 + 格雷码做跨时钟域处理。
```

### 2. 指定编码风格

```markdown
请使用以下风格：
- 所有组合逻辑用 always_comb 或 assign
- 所有时序逻辑用 always_ff @(posedge clk or negedge rst_n)
- 不要使用 initial 块（除 testbench 外）
- 所有参数用 parameter 声明，放在模块接口前
- 每个 always 块只驱动一个信号
```

### 3. 分层设计

复杂模块分步请求，而不是一次性：

```markdown
先写最顶层的接口定义和 FSM 状态机，确认无误后再实现各子模块。
```

## 常用 Prompt 模板

### 模板 1：组合逻辑模块

```
写一个 [模块名] 组合逻辑模块：
- 输入：[信号列表和位宽]
- 输出：[信号列表和位宽]
- 功能：[详细描述真值表或逻辑关系]
- 使用 assign 或 always_comb
```

**示例**：
```
写一个 priority_encoder 组合逻辑模块：
- 输入：[7:0] req
- 输出：[2:0] grant, valid
- 功能：优先编码器，req[7] 优先级最高，输出最高有效位的二进制编码，
  无请求时 valid=0
```

### 模板 2：时序逻辑模块

```
写一个 [模块名] 时序逻辑模块：
- 时钟：clk（[频率]）
- 复位：[异步/同步] rst_n，[高/低]有效
- 输入：[信号列表]
- 输出：[信号列表]
- 功能：[详细描述状态机或数据通路行为]
- 注意：[可综合性要求、时序收敛目标等]
```

### 模板 3：Testbench 生成

```
为 [模块名] 写一个 testbench：
- 时钟频率：[值]
- 测试用例：
  1. [用例1描述]
  2. [用例2描述]
  3. [用例3描述]
- 生成 VCD 波形文件
- 使用 $display 打印关键信号和断言结果
- 包含超时保护和测试结果汇总
```

### 模板 4：代码审查与优化

```
审查这段 Verilog 代码：
1. 是否存在隐含 latch？
2. 关键路径在哪里？如何优化？
3. 是否存在多驱动问题？
4. 代码是否符合可综合风格？
5. 给出具体的修改建议和优化后的代码
```

## 进阶技巧

### 上下文文件引用

Claude Code 可以直接读取项目中的文件：

```
参考项目中 rtl/arbiter.v 的编码风格，
用相同的接口规范 写一个 round-robin arbiter
```

### 增量修改

```
在这个 FSM 中增加一个 IDLE 状态和对应的跳转条件，
IDLE 状态等待 start=1 后进入 WORK 状态
```

### 跨文件操作

```
在 rtl/ 目录下新建 top.v，将现有的 pwm.v、uart.v 实例化进去，
并连接所有需要的接口信号
```
