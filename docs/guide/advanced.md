# 高级技巧

进阶使用 Claude Code 进行复杂 Verilog 工程开发的方法。

## 复杂模块设计

### 流水线设计

对于需要高吞吐的设计，让 Claude 帮你做流水线拆分：

```
将这段组合逻辑改成 3 级流水线：
- 第一级：[操作描述]
- 第二级：[操作描述]
- 第三级：[操作描述]
每级之间插入寄存器，计算最大工作频率。
```

### AXI/AXI-Stream 接口

```
写一个 AXI-Stream 数据打包模块：
- 从 AXI-Stream Slave 接收任意字节数的数据
- 打包成 32 字节的对齐帧输出到 AXI-Stream Master
- 处理 backpressure 和 tlast 信号
- 参数化最大包长度
```

### CDC（跨时钟域）处理

```
写一个参数化的 CDC 同步器：
- 支持单 bit（脉冲同步器）和多 bit（异步 FIFO）
- 正确处理亚稳态
- 包含约束文件模板（SDC/XDC）
```

## 验证框架搭建

### 自动生成 UVM 骨架

```
为这个模块生成一个 UVM 验证框架：
- uvm_sequence_item（transaction）
- uvm_sequence（激励生成）
- uvm_driver（驱动到 DUT）
- uvm_monitor（采集输出）
- uvm_scoreboard（比对参考模型）
```

### 覆盖率收集

```
给这个 testbench 增加功能覆盖率收集：
- 覆盖所有状态转换
- 覆盖边界条件（空/满等）
- 覆盖 corner case
```

## 综合与时序优化

### 综合约束引导

在设计时就让 Claude 考虑综合约束：

```
这个模块目标频率 250MHz，Virtex-7 器件。
请在代码中加入适当的流水线级，避免关键路径过长。
对于大位宽的组合逻辑，主动建议拆分方案。
```

### 面积优化

```
这个模块面积超标，请优化：
- 共享重复的计算单元
- 用资源替代逻辑（如 BRAM 替代寄存器堆）
- 串行化并行操作以节省面积
```

## 多文件工程管理

### 自动生成文件列表

```
扫描 rtl/ 目录下所有 .v 文件，生成一个综合用的 filelist.f，
正确处理层级依赖关系（先列出被依赖的模块）
```

### 自动生成顶层连线

```
读取 rtl/ 下所有模块的端口定义，生成一个 top.v，
将所有模块实例化并按照信号名自动互连
```

## 实战案例

### SPI Master 控制器

```
写一个 SPI Master 控制器：
- 支持 CPOL/CPHA 四种模式
- 参数化 SCLK 分频
- 支持 8/16/32 bit 传输
- 带 TX FIFO 和 RX FIFO
- AXI-Lite 寄存器接口
- 生成完整的文档注释
```

### 简单 RISC-V 核

```
写一个单周期 RISC-V RV32I 处理器：
- 支持基础整数指令集
- 哈佛结构（分开的指令和数据存储器）
- 先写出数据通路图（文字描述），确认后再逐模块实现：
  1. PC 和指令存储器
  2. 寄存器堆
  3. ALU
  4. 控制单元
  5. 数据存储器
```
