# 基础工作流

本章介绍使用 Claude Code 进行 Verilog 开发的日常工作流。

## 核心流程

```
头脑风暴 → 描述需求 → Claude 生成代码 → 审查与优化 → 仿真验证 → 迭代完善
```

## 0. 头脑风暴：先问清楚再动手

在描述具体需求之前，让 Claude Code 跟你一起头脑风暴。**不要上来就写代码**，而是让它问问题，帮你把需求想透。

### 为什么重要

- 你脑中想的 vs 你说出来的 vs 你写下来的，通常有差距
- Claude Code 通过提问帮你发现遗漏的边界条件、时序约束、接口协议
- 5 分钟的碰撞能省下后面 2 小时的返工

### 怎么触发

每次提需求时，主动要求 Claude Code 跟你头脑风暴：

```
我：我想做一个 AXI-Stream 数据包拆分模块，先别写代码，跟我头脑风暴一下

Claude：[开始提问]
  - 输入数据包的格式是什么？定长还是变长？
  - 拆分依据是什么？按字节数还是特殊标记？
  - 背靠背连续包怎么处理？
  - 目标频率多少？面积敏感还是吞吐优先？
  ...
```

### 典型问题清单

Claude Code 通常会从这些维度跟你确认：

- **接口协议**：AXI/SPI/UART/自定义？具体信号列表和时序要求
- **数据流向**：输入从哪里来，输出到哪里去，中间有没有存储/缓冲
- **边界条件**：空状态、满状态、异常输入、复位后的初始行为
- **性能目标**：频率、吞吐量、延迟、资源预算
- **可综合性**：目标 FPGA/ASIC、综合工具、代码风格约束

## 1. 描述需求

头脑风暴完成后，向 Claude Code 清晰地描述你的模块需求。一个好的需求描述包含：

- **接口定义**：输入输出信号及其位宽
- **功能行为**：模块应该做什么
- **时序约束**：时钟域、复位方式、延迟要求
- **特殊要求**：可综合风格、面积/速度优先等

### 示例对话

```
我: 写一个 8 位 PWM 控制器，参数 duty_cycle 控制占空比，
    输入 clk, rst_n, [7:0] duty_cycle，输出 pwm_out

Claude: [生成完整的 PWM 控制器代码，包含注释和参数说明]
```

## 2. 审查生成的代码

Claude 生成的代码通常质量很高，但你应关注：

- **可综合性**：代码风格是否适合目标器件
- **未连接信号**：是否有 latch 隐患
- **时序路径**：关键路径是否合理
- **复位逻辑**：异步/同步复位是否正确处理

## 3. 让 Claude 写 Testbench

代码生成后，继续让 Claude 帮你写仿真验证：

```
我: 给这个 PWM 模块写一个 testbench，用 100MHz 时钟，
    测试 25%、50%、75% 三种占空比
```

## 4. 运行仿真

```bash
# 编译
iverilog -o tb.vvp pwm.v tb_pwm.v

# 运行
vvp tb.vvp

# 查看波形（如果生成了 VCD）
gtkwave dump.vcd
```

## 5. 迭代优化

如果仿真不符合预期，把错误信息或波形截图发给 Claude，它能帮你定位问题并修复代码。

## 常用命令

在 Claude Code 会话中，你始终可以：

```
# 让它运行仿真
请用 iverilog 编译并运行这个 testbench

# 让它检查代码质量
检查这段代码的可综合性，有没有隐含 latch？
```

## 项目结构建议

```
my_verilog_project/
├── rtl/           # 设计源文件
│   ├── pwm.v
│   └── counter.v
├── tb/            # Testbench
│   ├── tb_pwm.v
│   └── tb_counter.v
├── sim/           # 仿真输出（.vcd 波形）
└── README.md
```
